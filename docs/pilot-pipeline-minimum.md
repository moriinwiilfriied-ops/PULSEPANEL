# Pipeline pilot — Minimum

Statuts simples pour suivre les leads et pilotes à la main (pas de CRM logiciel). Chaque statut a une définition, un next step et une preuve attendue.

---

| Statut | Définition | Next step | Preuve attendue |
|--------|------------|-----------|-----------------|
| **Lead identifié** | Entreprise / contact repéré, correspond à l’ICP. | Envoyer message (DM ou email). | Fiche prospect remplie (nom, contact, verticale, angle test). |
| **Message envoyé** | Premier message (LinkedIn ou email) envoyé. | Attendre réponse ou relancer après 3–5 j. | Copie du message ou note « envoyé le DD/MM ». |
| **Réponse reçue** | Le prospect a répondu (intérêt ou question). | Proposer call découverte 15 min. | Réponse sauvegardée ou résumée. |
| **Call booké** | Créneau de call découverte fixé. | Préparer démo, faire le call. | Date/heure + lien cal. |
| **Démo faite** | Call découverte ou démo réalisé. | Envoyer récap + proposition pilot. | Note : besoins, objections, cas d’usage retenu. |
| **Pilot proposé** | Offre pilot envoyée (prix, contenu, CTA). | Relancer si pas de réponse sous 3–5 j. | Email ou doc envoyé. |
| **Pilot signé** | Accord client (email, devis signé, paiement selon process). | Onboarder l’entreprise (checklist onboarding). | Accord écrit ou paiement reçu. |
| **Campagne lancée** | Au moins une campagne active ou terminée côté client. | Suivre KPI, attendre fin de campagne. | Campagne visible dans l’admin / dashboard. |
| **Proof capturée** | Résumé preuve généré, capture ou copie faite. | Préparer case study brouillon, demander citation. | Résumé preuve (dashboard) ou export. |
| **Repeat demandé** | Proposition V2 ou nouveau test envoyée au client. | Attendre réponse, fixer date relance. | Email ou note. |
| **Intro demandée** | Demande d’intro vers une autre boîte envoyée. | Noter si oui/non, relancer si besoin. | Réponse client. |

---

## Règles d’usage

- **Un prospect = une ligne** (ex. dans le CSV `docs/pilot-prospects-template.csv` ou outil externe).
- **Mettre à jour le statut** dès que l’étape est faite.
- **Preuve** : le minimum pour ne pas perdre l’info (date, copie courte, lien campagne si applicable).
- **Pas de sous-statuts** : rester sur cette liste pour éviter la complexité.

---

## Synthèse visuelle (ordre)

Lead identifié → Message envoyé → Réponse reçue → Call booké → Démo faite → Pilot proposé → Pilot signé → Campagne lancée → Proof capturée → Repeat demandé → Intro demandée.
