# Checklist — Après campagne (preuve + repeat)

À suivre une fois une campagne pilot terminée pour capturer la preuve, demander citation/logo/intro et proposer une V2.

---

## 1. Campagne terminée

- [ ] Statut campagne = **completed** (ou quota atteint).
- [ ] Client a bien vu le résultat (dashboard, export si besoin).
- [ ] Aucun litige en cours (qualité, retraits, flags) qui empêcherait d’utiliser la campagne en preuve.

---

## 2. KPI clés relevés

- [ ] **Volume** : réponses obtenues / quota.
- [ ] **Rapidité** : temps pour quota (si disponible).
- [ ] **Qualité** : % valides (bloc Qualité ou Résumé preuve).
- [ ] **Coût réel** : coût par réponse (détail campagne ou Résumé preuve).
- [ ] **Chiffre fort** : celui affiché dans le Résumé preuve (ex. « 100 réponses en 4 h », « 92 % valides »). Noter pour case study et commercial.

---

## 3. Proof pack capturé

- [ ] Page **détail campagne** ouverte (Dashboard → Campagnes → [nom]).
- [ ] Bloc **Résumé preuve** visible ; **Copier le résumé (Markdown)** utilisé et collé dans un doc ou sauvegardé.
- [ ] Capture d’écran du Résumé preuve (optionnel mais utile pour slide ou doc).
- [ ] Export CSV des réponses fait si besoin pour archive ou back-office.

---

## 4. Case study brouillon préparé

- [ ] Template utilisé : `docs/case-study-template.md`.
- [ ] Sections 1–4 remplies (problème, campagne lancée, résultats, décision prise) avec les vrais chiffres du Résumé preuve.
- [ ] Chiffre fort et résumé court insérés. Brouillon prêt à être validé par le client avant toute diffusion.

---

## 5. Citation demandée

- [ ] Demande envoyée au client (email ou call) : une phrase courte utilisable sur le site / en commercial.
- [ ] Accord écrit (email ou doc) reçu avant toute utilisation.
- [ ] Référence : `docs/proof-capture-and-client-reference-runbook.md` (section citation).

---

## 6. Logo demandé

- [ ] Demande d’autorisation logo (usage : site, one-pager, présentation) envoyée.
- [ ] Accord écrit reçu si oui ; sinon noté « refusé » ou « à redemander ».

---

## 7. Intro demandée

- [ ] Demande d’intro vers une autre boîte / personne du réseau envoyée (si relation consolidée).
- [ ] Réponse notée (oui / non / plus tard). Si oui, relance prévue.

---

## 8. Proposition V2 / nouveau test prête

- [ ] Client informé qu’il peut **dupliquer** la campagne (bouton « Créer une V2 ») pour relancer un test avec les mêmes paramètres ou les ajuster.
- [ ] Proposition concrète envoyée si pertinent : « On enchaîne sur une V2 avec [changement] ? » ou « Une autre campagne sur [sujet] ? »

---

## 9. Date de relance fixée

- [ ] Date ou échéance notée pour : relance citation/logo si pas de réponse, relance intro, ou proposition V2/autre campagne.
- [ ] Statut pipeline mis à jour (Proof capturée, Repeat demandé, Intro demandée selon le cas).

---

**Références** : `docs/case-study-template.md`, `docs/proof-capture-and-client-reference-runbook.md`, `docs/campaign-duplication-repeat-runbook.md`, `docs/lead-to-repeat-runbook.md`.
