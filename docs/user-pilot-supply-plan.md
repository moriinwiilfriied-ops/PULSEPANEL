# Plan de recrutement user pilot — PulsePanel

Plan minimal pour alimenter les premiers pilotes avec 200 à 500 users propres, sans acquisition sale.

---

## 1. Objectif chiffré

- **Cible** : 200 à 500 users pilot **propres** (réels, un compte par personne dans l’idéal, comportement cohérent avec le produit).
- **Horizon** : aligné sur le lancement des 5 à 10 premières entreprises et 10 à 30 campagnes. Pas de course au volume.
- **Qualité > quantité** : un user qui répond correctement et ne génère pas de flags vaut mieux que dix comptes douteux.

---

## 2. Pourquoi on ne cherche PAS du volume sale

- **Fraude** : volume massif trop tôt = multi-comptes, réponses bidon, retraits abusifs. Les garde-fous (flags, daily limits, gel retraits, devices) sont faits pour limiter la casse, pas pour encaisser une armée de fraudeurs.
- **Pilot crédible** : les entreprises pilotes ont besoin de **réponses exploitables**. Trop de réponses invalides (too_fast, empty) ou de comportement suspect = campagne inutilisable et perte de confiance.
- **Coût** : chaque user consomme du crédit (reward). Des users propres = meilleur ROI par campagne.
- **Réputation** : un pilot propre permet de demander preuve, citation, intro. Un pilot sale = rien à montrer.

---

## 3. Canaux prioritaires

| Canal | Pourquoi | Volume attendu | Contrôle |
|-------|----------|----------------|----------|
| **TikTok organique** | Court, clair, cible jeune/étudiants. Pas de budget ads. | Petit à moyen (50–150) | Contenu honnête, CTA simple. |
| **Instagram Reels** | Similaire, communautés niches. | Petit à moyen (50–150) | Idem. |
| **Petites communautés** | Forums, groupes FB, Discord ciblés (étudiants, side hustle, micro-tâches). | 20–80 par communauté | Message adapté, pas de spam. |
| **Étudiants** | Besoin de micro-revenus, réactifs. | 50–200 | Surveiller qualité (trop rapide, multi-compte). |
| **Groupes Discord** | Communautés thématiques (gaming, créa, études). | 20–50 par groupe | Accord admin, message court. |
| **Micro-influenceurs** | Petit levier (1–10 K abonnés), message authentique. | 30–100 par partenaire | Brief clair, pas de promesse de gains délirants. |

---

## 4. Canaux à éviter au début

- **Pub massive (Meta, TikTok ads)** : volume non maîtrisé, cible trop large, risque fraude élevé. Plus tard quand le process qualité est rodé.
- **Gros giveaways / concours** : attirent les chasseurs de bonus, multi-comptes, comportement one-shot. Pas de qualité.
- **Parrainage / referral** : à éviter tant qu’on n’a pas une base stable et des garde-fous clairs. Sinon boucle de comptes fictifs.
- **Annonces type “gagne 500 € / jour”** : attirent la fraude et décrédibilisent le produit.

---

## 5. Rythme recommandé

- **Phase 1** : 2–4 sources test (ex. 1 Reel, 1 Discord, 1 micro-influenceur). Lancer, mesurer qualité (flags, too_fast, empty, retraits) sur 7–14 jours. Ajuster ou couper.
- **Phase 2** : Garder les sources qui performent (qualité OK), en ajouter 2–3. Objectif 100–200 users en 4–6 semaines.
- **Phase 3** : Monter en volume sur les bonnes sources jusqu’à 200–500 users. Ne pas forcer : si une source dégrade la qualité, la couper.
- **Pas de sprint** : mieux vaut 300 users propres que 800 users dont 400 à surveiller.

---

## 6. Critères de qualité d’un user pilot utile

- **Un compte par personne** (pas de multi-compte évident ; devices partagés = signal à surveiller).
- **Réponses exploitables** : pas de flood too_fast / empty ; % valides des campagnes restent corrects.
- **Comportement cohérent** : répond, consulte le wallet, demande un retrait dans les clous (pas 50 demandes le premier jour).
- **Pas de pattern fraude** : pas de flags répétés, pas de gel nécessaire après revue.
- **Source tracée** : savoir d’où vient le user (quelle campagne de recrutement, quel canal) pour couper les mauvaises sources.
