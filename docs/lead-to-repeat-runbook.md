# Runbook — Du lead au repeat

Enchaînement opérationnel pour qualifier un lead, signer un pilot, livrer le résultat, capturer la preuve et pousser le repeat + intro. Court, concret.

---

## 1. Comment qualifier un lead

- **ICP** : voir `docs/pilot-icp-and-segmentation.md`. Bon prospect = petite marque / e-commerce / agence, parle de « tester » ou « valider », a un budget (même limité), décisionnaire ou proche.
- **Signaux à éviter** : panel France représentatif, rapport d’étude type institut, pas de budget, process achat long.
- **Action** : premier message (DM ou email) avec `docs/outreach-templates-pilot.md`. Si réponse positive ou curieuse → proposer call 15 min. Si pas de réponse → relance 1 puis 2 (espacées 3–5 j et 7–10 j).

---

## 2. Comment proposer le pilot

- **Offre** : `docs/pilot-offer-one-pager.md`. Promesse = une campagne de test simple, résultat en 24–72 h, dashboard + export + résumé preuve.
- **Prix** : forfait pilot (ex. 299–499 € HT) ou crédit consommable (100–300 €). À adapter au positionnement.
- **CTA** : « On configure ensemble la première campagne, vous lancez, premier résultat sous 72 h. »
- **Démo** : si besoin, démo 20 min avec `docs/outreach-templates-pilot.md` (script démo) et `docs/demo-script-10min.md`. Ne pas promettre de fonctionnalités non prêtes.

---

## 3. Comment onboarder vite

- **Checklist** : `docs/checklist-onboard-pilot-company.md`.
- **Séquence** : compte/accès validé → org liée → campagne (seed ou nouvelle) définie → budget/topup validé → cas d’usage, quota, reward validés → date lancement fixée → point de contact désigné → preuve/KPI attendus rappelés.
- **Blocage fréquent** : crédit non rechargé ou org mal liée. Vérifier Billing et org avant le jour J.
- **Objectif** : client opérationnel en 30 min à 1 h.

---

## 4. Comment lancer une première campagne simple

- **Checklist** : `docs/checklist-launch-campaign.md`.
- **Règles** : login + org OK, crédit suffisant, campagne relue (question, options, quota, coût), puis activation. Après activation : débit crédit, campagne en diffusion.
- **Support** : guider le client sur la première campagne (nom, question, options, quota) si besoin. Rester sur un cas simple (A/B 2 options, ou prix 4 paliers, ou slogan 3 options).

---

## 5. Comment lire le résultat

- **Dashboard** : détail campagne = réponses, progression quota, qualité (valides / trop rapide / vides), coût réel par réponse.
- **KPI** : `docs/pilot-kpi-minimum-spec.md`. Chiffres utiles = volume (réponses / quota), temps pour quota (si atteint), % valides, coût/réponse.
- **Résumé preuve** : bloc sur la page détail campagne (campagnes terminées) = résumé court + chiffre fort + bouton « Copier le résumé (Markdown) ». C’est la base pour case study et commercial.

---

## 6. Comment demander preuve + citation

- **Checklist** : `docs/checklist-post-campaign-proof-repeat.md`.
- **Preuve** : capturer le Résumé preuve (copie Markdown + capture écran). Préparer le brouillon case study avec `docs/case-study-template.md`.
- **Citation** : demander une phrase courte utilisable sur le site / en commercial. Accord écrit obligatoire avant utilisation. Détail : `docs/proof-capture-and-client-reference-runbook.md`.
- **Logo** : demander l’autorisation d’usage (site, one-pager). Accord écrit si oui.

---

## 7. Comment pousser une V2

- **Produit** : le client peut cliquer « Créer une V2 » sur la page détail campagne. La copie est en brouillon (paused), il peut modifier puis publier. Voir `docs/campaign-duplication-repeat-runbook.md`.
- **Commercial** : proposer explicitement « On enchaîne sur une V2 avec [X] ? » ou « Une autre campagne sur [sujet] ? ». Fixer une date de relance si le client dit « plus tard ».
- **Repeat** : une org avec au moins 2 campagnes = repeat positive. Suivi dans les KPI pilot (dashboard home, admin).

---

## 8. Comment demander une intro

- **Quand** : après résultat livré, et si possible après citation/logo OK. Relation consolidée.
- **Comment** : demander si le client peut recommander PulsePanel à une personne de son réseau (nom facultatif). Rester sobre : une intro = une opportunité de contact, pas une garantie de vente.
- **Ne pas forcer** : si refus, noter et éventuellement relancer plus tard. Référence : `docs/proof-capture-and-client-reference-runbook.md`.
